\begin{abstract}
We present TARDUS, a payment protocol that hides senders, receivers, and the transaction graph on Solana while leaving per-coin denominations as the only on-chain quantity leak. The design recovers David Chaum's 1982 blind-signature bearer cash idea and pairs it with four decades of follow-up work: Pointcheval--Stern unforgeability, Stinson--Strobl threshold blind signing, Komlo--Goldberg DKG, and the Dold 2019 cut-and-choose refresh. Where prior privacy efforts on Solana lean on succinct proofs, we deliberately avoid pairings, per-circuit trusted setup, and the 900\,k+ compute-unit cost those bring; the trade-off is that anonymity sets remain per-denomination rather than fully encrypted, and one extra off-chain round-trip per spend is required. We describe the on-chain bearer model, a threshold mint with HSM-resident shares, a four-layer fee-payer privacy stack that decouples the spend signer from any specific funding source, and a sealed-box AEAD payload format that mirrors NaCl \texttt{crypto\_box\_seal} but binds to Ed25519 receiving identities via the RFC 7748 birational map. We state nine security theorems, with proof sketches for T1--T7 and T9 plus an honest placeholder for T8, which becomes meaningful only after a Token-2022 ConfidentialMint vault migration that we describe as future work. The protocol has been deployed and exercised end-to-end on Solana devnet across the six core instructions; per-instruction compute-unit measurements range from 4\,k to 77\,k. The fee-payer privacy claim is validated empirically by an on-chain demonstration in which a recipient wallet receives native SOL through a spend whose transaction signer has no on-chain edge to the recipient's funding wallet.
\end{abstract}

\textbf{Keywords:} blind signatures; threshold signatures; bearer cash; payment privacy; cut-and-choose; sealed-box AEAD; Solana.
